# Create Event

`UiPath.MicrosoftOffice365.Activities.Calendar.CreateEventConnections`

Creates a calendar event in Microsoft Office 365. Supports attendees, recurrence, attachments, online meetings, importance, categories, sensitivity, and show-as status.

| Property | Value |
|---|---|
| **Package** | `UiPath.MicrosoftOffice365.Activities` |
| **Category** | Calendar |
| **Connector** | `uipath-microsoft-outlook365` |

## Input Properties

| Property | Display Name | Category | Type | Required | Default | Description |
|---|---|---|---|---|---|---|
| `CalendarArgument` | Calendar | | [`CalendarArgument`](components/CalendarArgument.md) | No | | Specifies the calendar where the event will be created. See [CalendarArgument](components/CalendarArgument.md) for input modes. |
| `Title` | Title | Input | `InArgument<string>` | Yes | | The name of the event. |
| `StartDateTime` | Start Date Time | Input | `InArgument<DateTimeOffset>` | Yes | | The date and time when the event starts. |
| `EndDateTime` | End Date Time | Input | `InArgument<DateTimeOffset>` | Yes | | The date and time when the event ends. |
| `Timezone` | Timezone | Input | `InArgument<string>` | Yes | | The timezone for the event's start and end time. |
| `Recurrence` | Recurrence | Input | `InArgument<string>` | No | | The recurrence details of the event. |
| `AllDayEvent` | All Day Event | Input | `InArgument<bool>` | No | `False` | Indicates if event takes place all day. Supersedes Start and End time. |
| `RequiredAttendees` | Required Attendees | Options | `InArgument<IEnumerable<string>>` | No | | The list of required attendees. |
| `OptionalAttendees` | Optional Attendees | Options | `InArgument<IEnumerable<string>>` | No | | The list of optional attendees. |
| `ResourceAttendees` | Resource Attendees | Options | `InArgument<IEnumerable<string>>` | No | | The list of resource attendees. |
| `Location` | Location | Options | `InArgument<string>` | No | | The location of the event. |
| `MeetingUrl` | Meeting URL | Options | `InArgument<string>` | No | | The link of the online meeting (Zoom, Teams, etc.). |
| `IsOnlineMeeting` | Is Online Meeting | | `InArgument<bool>` | No | `False` | Indicates whether this event should be created as an online meeting. |
| `Description` | Description | Options | `InArgument<string>` | No | | The description of the event. |
| `AttachmentInputMode` | Attachment Input Mode | Options | `AttachmentInputMode` | No | `Existing` | How to specify the attachments. |
| `Attachments` | Attachments | Options | `InArgument<IEnumerable<IResource>>` | No | | The attachment(s) to include with the event. |
| `ArgumentAttachments` | Attachments (Builder) | Options | `IEnumerable<InArgument<IResource>>` | No | | Individual attachment arguments. |
| `Importance` | Importance | Options | `InArgument<Importance>` | No | `Normal` | The importance of the event. |
| `Categories` | Categories | Options | `InArgument<IEnumerable<string>>` | No | | The categories to set on the event. |
| `ShowAs` | Show As | Options | `InArgument<FreeBusyStatus>` | No | `Busy` | The event status displayed in the calendar. |
| `Sensitivity` | Sensitivity | Options | `InArgument<Sensitivity>` | No | `Normal` | The sensitivity label applied on the event. |
| `PreferredReturnTimezone` | Preferred Return Timezone | Options | `InArgument<string>` | No | | Return event in the preferred timezone. |

| Property | Display Name | Category | Type | Description |

|---|---|
| `Importance` | `Low`, `Normal`, `High` |
| `FreeBusyStatus` | `Free`, `Tentative`, `Busy`, `Oof`, `WorkingElsewhere`, `Unknown` |

| `AttachmentInputMode` | `Existing`, `Builder` |

## XAML Example

```xml
<calendar:CreateEventConnections
    DisplayName="Create Event"
    Title="Team Meeting"
    StartDateTime="{x:Null}"
    EndDateTime="{x:Null}"
    Timezone="Eastern Standard Time"
    AllDayEvent="False"
    Importance="Normal" />
```

## Notes

- `Title`, `StartDateTime`, `EndDateTime`, and `Timezone` are required.
- Batching is used for performance when there are no attachments and no `Result` binding.
- The description body is always treated as HTML.
